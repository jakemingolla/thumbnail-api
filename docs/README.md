# Docs

Three doc sets. Each has a different reader and job.

## Agents

**Who:** coding agents and automation.
**So what:** executable context — how to navigate, change, and verify this repo without guessing.
**Language:** imperative, concrete, machine-scannable. Prefer paths, commands, and constraints over narrative.

## Human

**Who:** engineers and collaborators.
**So what:** orientation — why the system exists, how to work in it, and what “done” looks like for people.
**Language:** plain prose. Optimize for skimmability; assume a human will infer from examples.

## Specifications

**Who:** implementers (human or agent) who need a source of truth.
**So what:** contracts — behavior, interfaces, and invariants that must hold.
**Language:** precise and normative (`must` / `should`). Ambiguity is a bug; examples clarify, they do not replace rules.
